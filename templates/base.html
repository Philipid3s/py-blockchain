How PyBlockchain Works
Proof of Concept

PyBlockchain is a minimal blockchain built in Python that demonstrates how distributed ledger technology works as a content-sharing platform. It is intentionally small — the entire blockchain core is under 100 lines — so every concept is easy to follow and inspect.

Each post you write becomes a transaction. Transactions are bundled into a block and sealed with a cryptographic hash via Proof of Work. Once mined, data is permanent and tamper-evident.

The Chain Structure

Every block holds a hash of the previous block, forming a chain. Altering any block breaks every hash that follows — making the history immutable.

Proof of Work

Mining is the process of finding a nonce — an integer — such that:

SHA-256( index + transactions + timestamp + prev_hash + nonce )   starts with "00…" This is computationally expensive to find but trivial to verify. The number of required leading zeros is the difficulty (currently 2). Increasing difficulty makes mining exponentially harder.

Step-by-Step Flow
1
Create a Transaction
Fill in the form with your message and name. This calls POST /new_transaction and adds your post to the pending pool — it exists but is not yet on the chain.
2
Mine a Block
Click Request Mining. The server calls GET /mine, bundles all pending transactions into a new block, and runs the Proof of Work algorithm to find a valid hash. Once found, the block is appended to the chain and saved to SQLite.
3
Explore the Chain
Mined posts appear in the feed. Use Raw Chain to inspect the full JSON structure of the blockchain, or Pending TXs to see what is waiting to be mined. Data persists across restarts via the local blockchain.db file.

API Reference
Method Endpoint Description
GET / Main UI — loads and displays the chain
POST /new_transaction Add a transaction to the pending pool
POST /submit Form handler — wraps /new_transaction
GET /mine Run PoW, seal a block, append to chain
GET /chain Return full blockchain as JSON
GET /pending_tx Return unconfirmed transactions as JSON
POST /add_block Accept an externally mined block (node sync)
GET /reset Clear the entire chain and database
